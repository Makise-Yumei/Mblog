文件指定行列数据的批量读写

发表于2021-06-04|更新于2021-07-29|Python
|阅读量:

最近在尝试将风速当作工具变量时发现了几个很有意思的数据处理问题。

数据量大且杂乱、人工痕迹明显似乎不是什么新鲜的问题，但如何为未知格式的文件批量设置打开格式并进行读写，仍然鲜有较为全面的回答。

在这篇博客里，我将向你展示在文件格式未知的情况下，如何利用Python实现对文件指定行列数据的批量读写。

数据介绍

1.基本情况

这是2017年中国四百多个气象观测站点所提供的三百多万条地面气象数据。

  • 时间范围：大部分以三小时为单位，少部分有一小时数据。

  • 气象要素：气温、气压、露点、风向、风速、云量和降水量。

  • 数据格式：未知

2.处理目标

提取各站的年平均风速。

数据处理

1.数据描述

首先，让我们来观察一下文件的格式。

它是一个无法直接打开、必须通过记事板间接打开的文件，这将会在后续的数据读取中给我们带来麻烦。那么知道它的后缀名是否会对我们有所帮助呢？

可以看到，文件的后缀名是未知的。

关于如何获得文件后缀名的问题，我们可以使用Gnu。

安装完成后，在Gnu的bin目录下输入file [path] 即可查看文件格式。

很显然，知道后缀名——至少是这种形式的后缀名并不能为我们的数据处理减少麻烦。

也许，我们可以直接观察一下数据内容。

其中，第一列为年，第二列为月，第三列为日，第四列为小时，而第九列就是我们要提取的风速。每一列中或多或少出现的-9999则代表着缺失值。

看到这些数据，你第一时间想到的是什么呢？

没错，我们可以将它看作一个以空白为分隔符的csv文件。

那么接下来就是寻常的数据处理了。

2.代码实现

import os
import pandas as pd
import numpy as np

listx=[]
os.chdir('风速数据存储的绝对路径')
for filename in os.listdir(os.getcwd()): #遍历文件夹内所有文件
    listxappend(filename)

for i in range(412):
    f=pd.read_csv(listx[i],delim_whitespace=True,header=None) #以空白为分隔符读取文件
    f=f.replace(-9999,0) #处理缺失值
    a1=f.iloc[:,8] #指定第九列数据
    a2=np.mean(a1) #求第九列数据的平均值

print(a2)

运行代码，我们得到了这样的结果。

于是，我们成功地求出了2017年四百多个气象站的年平均风速。
