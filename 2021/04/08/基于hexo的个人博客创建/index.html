基于hexo的个人博客创建

发表于2021-04-08|更新于2021-07-29|hexo
|阅读量:

本文将向你展示使用hexo搭建个人博客的主要流程，在使用hexo的过程中所发现的错误将集中安排在另一篇文章hexo error集锦及解决方法中。

前期准备

1.下载Git

下载Git后，你就能够通过Git Bash进行各项操作，而不必使用cmd命令提示符。这将提高上传博客的稳定性。

2.下载Node.js

Hexo基本操作

1.安装

输入以下命令进行hexo安装：

npm install hexo-cli -g

之后，选择一个存放博客的目录：

hexo init blog
cd blog
npm install
hexo server

2.创建博文

hexo new "postname"

3.博客部署

hexo g
hexo s
hexo d

其中，第一行意为生成网站静态文件到默认设置的public文件夹。

第二行意为通过使用搭建在本地上的一个临时服务器预览博客页面。在输入该命令后，你将得到一个网址：http://localhost:4000，进入该网址即可预览你的博客。

但这不是永久的，且只有当你运行时该网页才能被访问。如果你想通过互联网查看自己的博客，你可以使用Github的静态页面。

第三行意为将你更新的博客资源上传到网站。当你成功绑定了GitHub之后，你就能够使用这一命令了。

连接Github

现在，你已经能够在临时服务器上生成自己的博客了。那么如何在互联网上查看自己的博客呢？这里我们将使用Github Pages。

1.在Github上新建一个名为_yourname.github.io_的仓库。

这里的_yourname_即为你在GitHub上的用户名。

2.修改_config.yml的deploy

打开_config.yml文件，将deploy后的代码改成如下内容：

deploy:
   type: git
   repo: url
   branch: gh-pages

在你建立于Github的博客仓库中，你可以通过点击右上角的code看到http地址，将它复制粘贴在上述代码的url中即可。

3.绑定你自己的域名（非必选）

这不是一个必要的步骤，使用默认的yourname.github.io来访问是完全可以的，但如果你希望拥有一个个性化的域名，那么你必须完成这一步。

  • 在Github下创建一个CNAME文件，在里面写上你的域名。
  • 找到Settings，在Github Pages的Custom domain里写上你的域名。

4.配置SSH keys

4.1生成新的SSH keys

 ssh-keygen -t rsa -C "youremail@youremail.com"

于是你得到了这样的回应：

Enter file in which to save the key:

Enter passphrase(empty for no passphrase):

第一行要求你选择密钥文件生成的地址及名称，跳过即表示使用默认设置。
第二行要求你设置密码，它会在你将来上传博客时使用，跳过则不用输入。

4.2将SSH keys与GitHub绑定

  • 打开id_rsa.pub文件（建议使用Notepad++），复制里面的内容。
  • 登录Github，找到Account Settings，点击SSH and GPG keys，将先前复制的密钥粘贴进SSH keys中。
  • 最后输入以下命令，检查SSH keys和github是否正常连接。
 ssh -T git@GitHub.com

主题

到目前为止，你已经成功创建了一个个人博客。也许你会觉得默认主题太过简单，那么如何更换主题，使你的博客更加个性化呢？

  • 挑选你喜欢的主题

访问hexo网站，挑选符合你喜好的主题。

  • 打开_config.yml文件，找到theme，将默认的landscape改为你挑选的主题。

  • 最后，在Git Bash里依次输入hexo g和hexo d，即可重新生成和发布你的博客。
